ESTRUCTURA SUGERIDA - LOGIN / REGISTER / RECUPERAR CONTRASEÑA + NAV PERSISTENTE
================================================================================

CONCEPTO CLAVE:
----------------
Lo que buscas (header/footer fijo mientras el contenido cambia) en Flutter
se llama "SHELL" o "layout persistente". Se logra con un widget que envuelve
el contenido y mantiene la barra de navegación inferior fija, mientras el
"body" central cambia según la pestaña seleccionada.

IMPORTANTE: Login, Register y Recuperar Contraseña normalmente NO llevan la
barra de navegación inferior (son pantallas previas al login). El "shell"
con nav bar aplica después de que el usuario inicia sesión (Home, Perfil, etc).
Si quieres, puedes compartir solo un header simple (logo/título) en las
pantallas de auth usando el mismo widget reutilizable.

NOTAS RÁPIDAS
--------------
1. Cada pantalla de auth (login, register, recover) es un archivo separado,
   cada una en su propia subcarpeta dentro de features/auth/.

2. Si más adelante login/register comparten piezas visuales (ej: un mismo
   campo de texto con validación, un mismo botón), esas piezas van en:
   features/auth/widgets/  (crea esa carpeta cuando la necesites)

3. app_shell.dart es la pieza que resuelve tu pregunta del "header/footer
   fijo mientras el contenido cambia". Ejemplo conceptual de cómo se ve por
   dentro (pseudo-código):

     Scaffold(
       appBar: AppHeader(),              // fijo arriba
       body: IndexedStack(                // cambia según la pestaña
         index: selectedIndex,
         children: [HomeScreen(), ProfileScreen(), ...],
       ),
       bottomNavigationBar: AppBottomNav( // fijo abajo
         currentIndex: selectedIndex,
         onTap: (i) => cambiarPestaña(i),
       ),
     )

4. Este árbol es una guía, no una regla estricta. Si tu equipo ya tiene
   convenciones distintas dentro de esas carpetas, sigue lo que ya exista
   para mantener consistencia con lo que hicieron tus compañeros.
